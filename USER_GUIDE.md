# Quackbase 用户使用说明

> 一个浏览器里就能用的数据探索平台。把 CSV / XLSX / ZIP 拖进来，就能像 Excel 一样浏览、筛选、搜索、分组统计、导出，但比 Excel 快得多，也能装得下千万级行数据。

## 一、能用它做什么

| 场景 | 怎么做 |
|---|---|
| 把一份大 CSV 丢进来快速看一眼 | 拖文件 → 选表 → 看表格 |
| 把一个项目下几十个表批量入库 | 把它们打成 ZIP → 一键上传 |
| 一周追加一次新数据到同一张表 | 选"追加到已有表"模式上传 |
| 在千万级数据中搜一段文字 | 用顶部搜索框 |
| 按时间范围 / 数值范围 / 文本筛选数据 | 用筛选条 |
| 按某一列汇总：每个城市卖了多少？ | 切到"分组统计"，选分组字段 + COUNT/SUM |
| 按两个维度交叉汇总：每个城市每个产品的销量 | 分组字段选 2 个 |
| 把筛选后的结果导出给别人 | 点导出按钮，自动下载 CSV |

支持的文件格式：

- `.csv` —— 自动识别 UTF-8 / GBK / GB18030 等编码，无需手动转码
- `.xlsx` —— 自动转成 CSV 入库；多 Sheet 表头一致时自动合并
- `.zip` —— 解压后递归找 csv/xlsx 批量入库

## 二、登录

浏览器打开访问地址（部署同事会告诉你），输入用户名 / 密码即可。

首次使用的默认账号：

| 用户名 | 密码 |
|---|---|
| `admin` | `admin123` |

> 登录后请立即点左侧用户栏的 🔑 修改密码。
>
> 服务端每次重启会重新生成签名密钥，所有已登录会话会失效，需要重新登录——这是正常行为。

## 三、上传数据

左侧上传区有两种方式：

- **点击** 上传区 → 弹出系统文件选择框
- **拖拽** 一个或多个文件到上传区

### 3.1 单个 CSV / XLSX 文件

1. 拖入 / 选中一个 `.csv` 或 `.xlsx`
2. 弹出"选择导入方式"窗口
3. 选 **新建表**（默认），可改表名（默认用文件名）
4. 点 **确认导入** → 表立刻出现在左侧表列表

### 3.2 批量上传多个文件

一次拖入多个 csv/xlsx，弹窗会列出所有文件，每个文件可单独：

- 修改表名
- 点 ✕ 移除不想导入的

支持勾选"忽略以 `#` 开头的注释行"——CSV 文件头部有说明文字时勾上即可。

### 3.3 ZIP 压缩包上传（推荐用于多文件 / 大文件）

**适用场景**

- 一次要上传几十个表 —— 打成 zip 比一个个拖快得多
- 单个 CSV 太大（≥ 500MB）—— 系统会**强制要求**先压缩成 zip 再上传

**操作步骤**

1. 把若干 csv / xlsx（可以放在子目录里）打包成 `xxx.zip`
2. 拖入上传区
3. 弹框会**立刻打开**显示 "正在解析 ZIP..."（无需等待）
4. 解析完成后展示清单：

   ```
   📦 来自 ZIP：报表.zip（5 个文件）
   ┌────────────────────────────────────────┐
   │ 📄 销售/华东.csv          表名: [销售_华东] │
   │ 📄 销售/华南.csv          表名: [销售_华南] │
   │ 📄 库存/2026Q1.xlsx       表名: [库存_2026Q1]│
   │ ...                                       │
   └────────────────────────────────────────┘
   ```

5. 每行可单独修改表名（默认用相对路径自动生成，避免同名冲突）
6. 点 **确认导入** → 所有文件依次入库，每张表入库结果以 toast 提示

> 如果弹框内容比屏幕高，可以在框内上下滚动；header（标题栏）和 footer（按钮）始终固定。

**ZIP 上传的规则**

| 情况 | 行为 |
|---|---|
| ZIP 内含 csv / xlsx 以外的文件（如 `.txt`、图片） | **整个 ZIP 拒绝**，需清理后重传 |
| ZIP 内有子目录 | 递归扫描，全部导入 |
| 不同子目录有同名文件（如 `dir1/a.csv` 和 `dir2/a.csv`） | 自动用相对路径区分表名（`dir1_a` vs `dir2_a`） |
| ZIP 内的 macOS 元数据（`__MACOSX/`、`.DS_Store`） | 自动跳过，不会报错 |
| ZIP 中某个文件解析失败（列结构有问题等） | 该文件失败，**其他文件继续导入**，弹窗各自提示 |
| 同时拖入 ZIP 和 csv 混合 | 拒绝，提示"一次只能上传一个 ZIP，且不能与其他文件混合" |
| ZIP 模式 | 强制 **新建表（replace）**；不支持追加模式 |

**为什么 CSV ≥ 500MB 必须压缩？**

- 上传更快（CSV 文本压缩率通常 70%+）
- 服务器接收上限是 500MB；超过会被网关拒绝
- 系统会前置拦截：选择文件时如果检测到 ≥ 500MB 的 CSV，会直接弹错"请先压缩为 ZIP 再上传"

### 3.4 多 Sheet 的 XLSX

上传单个 xlsx 时，如果文件里有多个 Sheet，会进入 **多 Sheet 导入预览**：

- 系统按 **表头是否一致** 自动分组（同表头多个 Sheet 会被建议合并成一张表）
- 每个分组可：
  - 勾选是否导入
  - 修改表名
  - 看到包含哪些 Sheet 和列名
- 点 **确认导入这些分组** 完成

### 3.5 上传模式：新建表 vs 追加到已有表

普通 csv/xlsx 上传时（非 zip），有两个互斥的模式：

| 模式 | 含义 |
|---|---|
| **新建表**（默认） | 每个文件创建独立的新表；如果同名表存在会被**整体替换** |
| **追加到已有表** | 把数据**追加**到指定的已有表中（要求两边列名完全一致） |

追加模式下，文件的列名必须和目标表一模一样，否则会报"列结构不匹配"，**不会**部分追加。

### 3.6 已上传文件管理

左侧上传区下方会显示**已上传文件**计数（如 `已上传文件 (12)`），点 **展开** 可查看：

| 操作 | 行为 |
|---|---|
| 鼠标移到文件行 → 点右侧 ✕ | 删除单个源文件，不影响已入库的表 |
| 点底部 **清除全部文件** | 清空 `uploads/` 下所有源文件 |

> 这里删的是**上传时落到服务器的原始 csv/xlsx/zip 文件**，不会动数据库里的表。
>
> 如果你不再需要重新追加 / 重新导入，可以放心清理释放磁盘空间。

## 四、浏览与查询数据

页面顶部从上到下依次是：

1. **应用栏**：菜单切换 / 表名 / 行数 / 删除按钮 / 导出按钮
2. **视图切换**：📋 数据 ↔ 📊 分组统计
3. **🔎 数据筛选 card**：筛选条件 + 全表搜索
4. **数据表 card** 或 **统计配置 / 统计结果 card**

### 4.1 切换数据表

左侧表列表显示所有已上传的表，每行有：

- 表名
- 行数
- 列数

点表名即可切到该表。

### 4.2 排序与列宽

- **排序**：点列头一次 → 升序，再点 → 降序，再点 → 取消。时间戳 / 数值 / 文本都按各自类型排序。
- **列宽调整**（分组结果表）：拖列头右边缘可调宽，**双击右边缘**重置为默认宽度。

### 4.3 筛选（按列条件）

#### 添加筛选

在 **数据筛选 card** 里点 **+ 添加筛选条件** → 弹出筛选弹框：

- 选列名
- 选操作符：`=` / `!=` / `>` / `<` / `>=` / `<=` / `LIKE` / `BETWEEN`（日期列）/ `IS NULL` / `IS NOT NULL`
- 输入值

多个条件之间是 **AND**（同时满足）。

#### 已有条件的快速编辑 / 删除

已添加的筛选条件会显示成黄色 **chip**：

| 操作 | 行为 |
|---|---|
| **点 chip 主体（带虚线下划线提示）** | 重新打开弹框，可直接修改这条条件 |
| 点 chip 右侧 ✕ | 删除该条件 |
| 顶栏点 **✕ 清除筛选** | 一次清空全部条件 |

#### 日期 / 时间筛选

对类型为 `DATE` / `TIMESTAMP` / `DATETIME` 的列：

- 弹框会自动展示 **日期选择器**（不再是文本框）
- `TIMESTAMP` 列精确到 **秒**
- 默认操作符是 `BETWEEN`（区间）；也可改为 `=` / `>=` 等
- 区间模式：选起止两端，起始默认 `00:00:00`、结束默认 `23:59:59`

```
列：支付完成时间
操作：BETWEEN
起：2026-04-15 00:00:00
止：2026-04-30 23:59:59
```

> 系统会自动把选中的字符串值转成 TIMESTAMP / DATE 与列做比较，无需写 SQL。BETWEEN 在内部拆成两条 `>=` 和 `<=` chip。

### 4.4 全局搜索

数据筛选 card 里的 **🔍 全表搜索** 输入框：

- 在 **所有文本列** 里做不区分大小写的模糊匹配
- 数值 / 时间列会跳过（避免无意义的 CAST 拖慢查询）
- 自动 400ms 防抖，停手才发请求
- 可以与筛选条件叠加 —— 先按时间筛选，再搜文字，结果是两者的交集

### 4.5 分页

底部分页栏可：

- 切到任意页
- 修改每页条数（默认 50）
- 看到总行数 / 总页数

> 翻页 / 改每页条数时，后端会**跳过总数 COUNT 重算**，只查当前页数据；只有筛选条件本身改了才会重算总数。这是几十毫秒和几秒的差别。

### 4.6 视图切换：数据 / 分组统计

数据筛选 card 上方有一对 tabs：

- **📋 数据** —— 显示原始数据表（默认）
- **📊 分组统计** —— 切到分组统计页（详见第五章）

切换视图时，已经设置的 **筛选条件和搜索词会保留**，两种视图共享同一份"当前数据范围"。

## 五、分组统计

按某列（或两列组合）汇总，并对其他列做 COUNT / SUM / AVG / MIN / MAX 计算。

切到 **📊 分组统计** 后，你会看到两张卡片：

- **📊 统计配置**：选分组字段、配置聚合项、点 **应用统计**
- **📈 统计结果**：表格 + 翻页 + 列宽拖拽

### 5.1 选分组字段（最多两个）

| 字段 | 说明 |
|---|---|
| **主分组列**（必选） | 必须先选 |
| **次级分组（可选）** | 选了才会出现；最多支持 2 个；与主分组不能重复 |

举例：

- 只选 `城市` → 每个城市一行
- 选 `城市` + `产品` → 每个 (城市, 产品) 组合一行

> 主分组清空时，次级会一起清空。

### 5.2 配置聚合项

每个聚合项有三个部分：

| 部分 | 选项 |
|---|---|
| **操作** | `COUNT 计数` / `SUM 求和` / `AVG 平均值` / `MIN 最小值` / `MAX 最大值` |
| **目标列** | COUNT 可以是 `*`（所有行）；SUM / AVG 只能选数值列；MIN / MAX 任意列 |
| **别名** | 结果表里的列名；不填会自动生成（如 `sum_金额`） |

支持 **多个聚合项**：

- 点 **+ 添加聚合项** 增加一行
- 每行右侧 ✕ 删除（最后一行不能删）

举例：

```
分组：城市
聚合：
  COUNT(*)        → count
  SUM(金额)        → 总销售额
  AVG(金额)        → 平均客单
```

结果表头：`城市 | count | 总销售额 | 平均客单`，每行一个城市。

### 5.3 与筛选 / 搜索的关系

统计配置 card 顶部会显示当前状态：

- `已应用 2 个筛选条件 + 全局搜索` —— 统计基于筛选后的数据
- `未应用任何筛选 / 搜索` —— 统计基于全表

修改筛选 / 搜索后**需要重新点 应用统计** 才会刷新结果。

### 5.4 排序、翻页、列宽

| 操作 | 行为 |
|---|---|
| 点列头 | 切换排序：默认按第一个聚合项降序，点列头切其他列升 / 降 |
| 拖列头右边缘 | 调列宽 |
| 双击列头右边缘 | 重置该列宽度 |
| 底部翻页 | 默认 50 行 / 页，与数据视图一致 |

> 翻页 / 改排序时，后端只查当前页，不重算分组总数 —— 速度很快。

### 5.5 导出分组结果

统计配置 card 右侧的 **↓ 导出 CSV**：

- 导出**当前页**结果（不是全部分组）
- 想导出更多结果，请先调大每页条数（最大 200）

> 完整分组结果导出能力计划中，目前可以先调大 page size 再导出。

## 六、导出数据

数据视图顶栏的 **↓ 导出 CSV** 按钮 → 浏览器自动下载。

| 选项 | 行为 |
|---|---|
| 当前已应用筛选 / 搜索 | 默认导出**筛选后**的数据 |
| 想导出全表 | 选 "导出全部数据"（忽略当前筛选） |

导出文件特性：

- 自带 UTF-8 BOM，Excel 打开不会乱码
- 流式下载，几百万行也不会卡浏览器
- 文件名自动带表名

## 七、删除数据

| 操作 | 入口 | 影响 |
|---|---|---|
| 删除整张表 | 表列表 → 表名右侧 ✕ | 该表及其数据全部消失，**不可恢复** |
| 按条件删除部分行 | 应用筛选条件后 → 顶栏 **✕ 删除筛选数据** | 只删除匹配的行，其他行保留 |
| 清空整张表的数据（保留表结构） | 无筛选时 → 顶栏 **✕ 清空表数据** | 表还在，数据为 0 行 |
| 清空全部上传过的源文件 | 左侧文件管理 → 清空 | 仅删 `uploads/` 下的源文件，**不影响已入库的表** |

> 删除是物理删除，没有回收站。重要数据请先导出备份。

## 八、用户与权限

| 角色 | 上传 | 查看 | 删除 | 用户管理 |
|---|---|---|---|---|
| **admin** | ✅ | ✅ | ✅ | ✅ |
| **editor** | ✅ | ✅ | ✅ | ❌ |
| **viewer** | ❌ | ✅ | ❌ | ❌ |

每个用户的数据是 **完全隔离** 的 —— 你只能看到你自己上传的表，admin 也不会越权访问别人的库。

**用户管理（仅 admin）**：

- 左侧 → 管理 → 用户管理
- 可以新增用户、改角色、删除用户
- 自己的账号会显示"（当前用户）"标记，不能在用户管理里删除自己 / 改自己的角色

**修改自己的密码**：

- 任何角色都可以
- 左侧用户栏 → 🔑 → 填原密码 + 新密码（至少 6 位）
- 修改成功后会自动登出，需要用新密码重新登录

## 九、常见问题

**Q1. 上传 CSV 中文乱码？**

系统会自动识别 GBK / GB18030 / UTF-8 等编码并转成 UTF-8 入库，正常情况不会乱码。如果出现问题，请确认原文件的实际编码——某些工具导出的"假 UTF-8"（实际是 ANSI）可能识别失败。

**Q2. 上传 xlsx 后只看到部分数据？**

xlsx 多 Sheet 时，系统只合并 **表头完全一致** 的 Sheet。表头不同的 Sheet 会被分到不同分组，多 Sheet 预览弹框里可以分别勾选导入。

**Q3. CSV 文件第一行不是表头怎么办？**

如果是 `#` 开头的注释行，勾选上传弹窗里的 **忽略注释行**。如果就是数据行混入了说明文字，请先在 Excel 里清理后再上传。

**Q4. 追加上传时报"列结构不匹配"？**

追加模式要求源文件和目标表的列名 **完全一致**（顺序可不同，但名字得一字不差）。即使多了一列或少了一列都会拒绝。建议要么改源文件、要么用"新建表"覆盖。

**Q5. 上传 ZIP 报"包含不支持的文件"？**

ZIP 里只能放 csv / xlsx。常见误入：

- README.txt / .md → 删掉
- 备份文件（如 `~$xxx.xlsx`，xlsx 编辑时的临时文件）→ 删掉
- 图片、PDF → 删掉

清理后重新打包再上传即可。

**Q6. 单 CSV 大小被拦截怎么办？**

这是设计行为：≥ 500MB 的 CSV 必须压缩成 ZIP 再上传。

```
原 CSV 1.2GB → 压成 zip 通常 200-400MB
```

打包方法：

- macOS：右键 → 压缩
- Windows：右键 → 发送到 → 压缩文件
- 命令行：`zip xxx.zip xxx.csv`

**Q7. 时间筛选报"TIMESTAMP and VARCHAR"错误？**

新的日期选择器已经自动处理类型转换，正常情况不会出现。如果你用更早版本的 chip 或 URL 手动改了筛选，请重新通过弹框选日期：

| 列类型 | 推荐格式 |
|---|---|
| TIMESTAMP | `2026-04-15 23:59:59` |
| DATE | `2026-04-15` |

**Q8. 分组统计点了"应用统计"后没反应 / 报错？**

按这个顺序排查：

1. 主分组字段是否已选？（必填）
2. 聚合项里的 SUM / AVG 是否选了**数值列**？（文本列会被拒绝）
3. 非 COUNT 操作是否指定了目标列？（不能留空 / `*`）
4. 错误信息会显示在配置 card 底部的红色横幅里

**Q9. 分组统计的数字显示成 `6.59e+7` 一样的科学计数法？**

新版已修复：所有结果数字按 **千分位** + 浮点保留 2 位小数显示（如 `65,934,210.00`）。如果你看到的还是科学计数法，刷新一下浏览器拿最新版前端即可。

**Q10. 删除数据后能恢复吗？**

不能。系统没有回收站。重要数据建议：

1. 上传前留好原始文件
2. 重要表定期 **导出 CSV** 做备份
3. 删除前再三确认 confirm 弹框里的提示

**Q11. 上传 ZIP 后弹框等很久才出现？**

新版已改为 **点上传后立即弹框** 并显示 "正在解析 ZIP..." 加载状态，无需等后端解析完。如果你仍然要等 → 刷新页面拿最新前端。

**Q12. 弹框内容超出屏幕怎么办？**

弹框已支持滚动：header（标题栏）+ footer（按钮栏）固定，中间内容区域可以上下滚动；最大高度限制在 `屏幕高度 - 48px`。
